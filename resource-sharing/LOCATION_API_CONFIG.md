# 地图定位配置指南

## 概述

本平台使用高德地图API实现精准的地理位置定位功能。系统支持两种定位模式：

1. **在线模式**（推荐）：使用高德地图API，提供准确的地址名称
2. **离线模式**（备用）：匹配预设的大学/校园地点列表

## 高德地图API配置

### 1. 获取API密钥

1. 访问 [高德开放平台](https://console.amap.com/dev/key/app)
2. 注册/登录账号
3. 创建应用，选择「Web端(JS API)」
4. 获取对应的Key（Web服务）

### 2. 配置API密钥

在 `src/components/RealtimeLocation.tsx` 文件中，找到第10行：

```typescript
const AMAP_KEY = '您的API密钥'; // 替换为实际的API密钥
```

将 `您的API密钥` 替换为您在高德开放平台获取的真实API密钥。

### 3. API密钥配置要求

- **密钥类型**：Web服务（Web Service）
- **IP白名单**：开发环境可留空或设置为 `*`
- **域名白名单**：开发环境可留空或设置为 `*`
- **配额**：个人开发者默认配额为 60万次/天，足够校园平台使用

## 功能说明

### 在线模式（高德地图API）

- **精度**：精确到POI（兴趣点）级别，如"海南大学·海甸校区"
- **响应**：返回详细的地址信息
- **性能**：依赖网络，响应时间约500ms-1s

**工作流程**：
1. 获取GPS坐标（经纬度）
2. 调用高德地图逆地理编码API
3. 将坐标转换为真实地址名称
4. 显示给用户

### 离线模式（备用方案）

- **精度**：匹配预设的15所大学，显示"距离XX大学X公里"
- **响应**：无需网络，即时响应
- **性能**：完全本地计算

**触发条件**：
- API密钥未配置
- 网络请求失败
- API调用超时

## 常见问题

### 1. 定位不准确怎么办？

- **检查**：确保已配置正确的高德地图API密钥
- **切换**：可以手动选择位置，点击"手动设置"按钮
- **权限**：确保浏览器允许定位权限

### 2. 为什么显示"离线模式"？

- API密钥未配置或格式错误
- 网络连接问题
- API服务暂时不可用

### 3. API调用配额不足？

高德地图个人开发者默认配额为60万次/天，一般不会超限。如果超限：
- 升级为企业账号
- 或使用离线模式作为备用

### 4. 如何测试定位功能？

1. 使用浏览器开发者工具模拟GPS位置：
   - Chrome: F12 → More tools → Sensors → Location
   - 选择"Custom"设置测试坐标

2. 真机测试：
   - 使用手机浏览器访问
   - 允许定位权限

## 安全性

- API密钥不应该暴露在客户端代码中
- 生产环境建议：
  - 使用环境变量存储API密钥
  - 或通过后端代理API请求
